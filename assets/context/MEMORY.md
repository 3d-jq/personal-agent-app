# MEMORY

跨场景长期记忆。

> 写入原则：只能记录用户明确陈述的事实，禁止推断、脑补，禁止写入用户未确认的分析或建议。
> 日期格式统一 `YYYY-MM-DD`。

## 长期稳定区（不易变化）

### 重要决策与结论

- [{YYYY-MM-DD}] {结论及理由}
- 说明：用户做出的重大决策和最终结论。一旦记录很少变动，除非用户推翻。

### 稳定事实

- {事实}（如生日、毕业学校、家庭成员等关键信息）
- 说明：长期不变的个人事实。

### 工具与备忘

- {备忘标题}：{值}
  - 创建：{YYYY-MM-DD}
- 说明：API key 位置、常用命令、联系人、特定项目入口等。

## 时效性区（会随时间变化）

### 进行中的项目

- {项目名}（{YYYY-MM-DD}）
  - 目标：{值}
  - 进展：{值}
  - 待办：{值}
- 说明：活跃项目的当前状态。完成或长期无更新时应归档或删除。

### 用户习惯与偏好

- {偏好}：{表现}
  - 观察起：{YYYY-MM-DD}
  - 备注：{触发场景}
- 说明：用户的行为模式、节奏倾向、表达习惯。

### 临时任务与待办

- {任务}（{YYYY-MM-DD}）
  - 状态：{...}
  - 截止：{...}
- 说明：短周期的待办、提醒。
