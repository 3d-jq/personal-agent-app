# 🍎 Apple 风格设计规范

> **适用对象**：personal-agent-app（Flutter / Android AI 助手）  
> **目标**：让用户在每次交互中都感受到 Apple 级别的精致、克制与一致性  
> **版本**：v1.0 | 2026-06-24

---

## 一、设计哲学

### 1.1 核心原则

| 原则 | 说明 | Apple 出处 |
|------|------|-----------|
| **清晰胜过炫技** | 每一个元素都有存在的理由，没有多余的装饰 | Human Interface Guidelines |
| **深度即层次** | 通过光影、模糊、缩放建立真实的视觉纵深 | iOS/macOS 设计语言 |
| **尊重用户注意力** | 动画服务于理解，不为了动而动 | WWDC 2023 - Designing for Focus |
| **一致性建立信任** | 同样的操作在相同场景下永远有相同的结果 | iOS HIG |

### 1.2 情感曲线

```
用户打开 App → 感受到稳定与熟悉 → 产生掌控感 → 对 AI 的建议产生信任
```

Apple 产品的所有设计决策都围绕这个曲线展开。你的 AI 助手也不例外。

## 二、视觉系统

### 2.1 色彩体系

#### 主色调

```
主色 (Primary):    #007AFF  (iOS System Blue)
成功色:            #34C759  (iOS System Green)
警告色:            #FF9500  (iOS System Orange)
错误色:            #FF3B30  (iOS System Red)
中性色阶:
  主文字:          #000000 / #FFFFFF (自适应深色模式)
  次要文字:        rgba(0,0,0,0.55) / rgba(255,255,255,0.55)
  分割线:          rgba(60,60,67,0.12) / rgba(84,84,88,0.12)
  背景:            #F2F2F7 (iOS System Gray 6) / #000000
  卡片背景:        #FFFFFF / rgba(28,28,30,0.95)
```

#### 语义化色彩规则

- **状态反馈**：绿色 = 完成、蓝色 = 信息/进行中、橙色 = 等待确认、红色 = 错误/阻断
- **AI 消息气泡**：始终使用主色 `#007AFF`，与用户消息形成层次区分
- **渐变使用**：仅在关键 CTA 按钮使用渐变，禁止大面积渐变背景

### 2.2 字体规范

#### 系统字体栈（全平台统一）

```yaml
中文: "PingFang SC", "SF Pro SC", -apple-system, "Helvetica Neue", sans-serif
英文: "SF Pro Display", "SF Pro Text", -apple-system, "Helvetica Neue", sans-serif
数字/代码: "SF Mono", "Menlo", monospace
```

#### 字号层级

| 层级 | 字号 | 字重 | 使用场景 |
|------|------|------|---------|
| Large Title | 34pt | Bold | 页面标题（极少用） |
| Title 1 | 28pt | Bold | 对话 AI 名称、关键区域标题 |
| Title 2 | 22pt | Bold | 卡片标题、设置页分组 |
| Title 3 | 20pt | Semibold | 消息时间戳、子标题 |
| Body | 17pt | Regular | **正文默认，AI 回复标准字号** |
| Callout | 16pt | Regular | 注释、辅助说明 |
| Subhead | 15pt | Regular | 列表项、表单标签 |
| Footnote | 13pt | Regular | 时间戳、版权信息 |
| Caption 1 | 12pt | Regular | 图标标签 |
| Caption 2 | 11pt | Regular | 辅助标记 |

**铁律**：正文字号永远 17pt，不允许在对话流中混用其他字号制造"重点"，重点通过颜色和间距表达。

### 2.3 间距系统

```
基础单位: 8pt

间距层级:
  4pt  (0.5x)   - 图标与文字、紧密相关的元素
  8pt  (1x)     - 列表项间距、文字行高
  12pt (1.5x)   - 卡片内部 padding
  16pt (2x)     - 卡片之间间距
  24pt (3x)     - 区域之间间距
  32pt (4x)     - 页面级间隔
```

**规则**：所有间距必须是 8pt 的整数倍，禁止出现 5pt、7pt、13pt 等值。

### 2.4 圆角与阴影

```
圆角半径:
  按钮: 20pt (iOS 系统按钮标准)
  卡片: 16pt
  输入框: 12pt
  消息气泡: 18pt (iOS iMessage 标准)
  底部面板: 28pt (大圆角 Sheet 标准)

阴影:
  层级 1 (悬浮): boxShadow(0 2 8 0 rgba(0,0,0,0.08))
  层级 2 (弹窗): boxShadow(0 8 32 0 rgba(0,0,0,0.12))
  禁止使用纯黑阴影，必须带透明度
```

## 三、组件规范

### 3.1 消息列表（核心组件）

```
┌─────────────────────────────────┐
│ [AI 头像]  AI 消息气泡             │
│           圆角 18pt，背景主色       │
│           内边距 12pt 16pt         │
│           文字白色，行高 1.5        │
│           最大宽度 80%             │
└─────────────────────────────────┘

用户消息：
┌─────────────────────────────────┐
│                       [用户头像]  │
│                       [用户气泡]  │
│                       圆角 18pt    │
│                       背景 #E9E9EB│
│                       文字深色      │
└─────────────────────────────────┘
```

**气泡对齐规则**：
- AI 消息：左对齐，头像在左，最大宽度 80%，距屏幕左侧 16pt
- 用户消息：右对齐，头像在右，最大宽度 80%，距屏幕右侧 16pt
- 连续同一发言者的消息：头像仅在第一条显示，后续消息减少左侧间距

### 3.2 工具调用展示

AI 调用工具时，在气泡下方插入一条轻量状态条：

```
┌─────────────────────────────────┐
│ 🔄 正在查询天气...                │  ← 状态条
│           字体 13pt，颜色次要文字  │
│           左侧小图标 + 动态文字   │
└─────────────────────────────────┘

完成后：
┌─────────────────────────────────┐
│ ✅ 天气查询完成（北京，26°C）      │  ← 已完成
│           颜色变为成功绿           │
└─────────────────────────────────┘
```

**规则**：
- 进行中：蓝色旋转图标 + "正在 xxx..."
- 成功：绿色对勾 + 结果摘要
- 失败：红色感叹号 + 错误原因 + 重试按钮
- 禁止用 Toast/Alert，全部内联展示

### 3.3 任务规划（Task Plan）可视化

```
┌─────────────────────────────────┐
│ 📋 任务规划                       │
│ ┌─ 分析需求                    ✓ │  ← 完成：带对勾，文字次要色
│ ├─ 搜索信息                  ⟳   │  ← 进行中：蓝色高亮，左侧蓝条
│ └─ 生成报告                        │  ← 待办：正常显示
└─────────────────────────────────┘
```

**状态表达**：
- `pending`：正常文字色，无额外装饰
- `in_progress`：文字蓝色 `#007AFF`，左侧 3pt 蓝色指示条，整行背景高亮 `rgba(0,122,255,0.06)`
- `completed`：文字次要色，带 `✓` 图标
- `failed`：文字红色，带 `✗` 图标，下方展示错误原因

**动画**：状态切换时使用 `AnimatedContainer`，过渡时长 200ms，曲线 `Curves.easeInOut`。

### 3.4 输入框

```
┌─────────────────────────────────┐
│  ┌─────────────────────────────┐│
│  │ 输入消息...              📎 ││  ← 始终显示附件按钮
│  └─────────────────────────────┘│
│            [发送] 按钮            │
└─────────────────────────────────┘
```

**规范**：
- 输入框高度：36pt（单行），最多 5 行后滚动
- 圆角：20pt
- 边框：1pt，颜色分割线色
- 聚焦态：边框变主色 `#007AFF`，0.5s 过渡
- 发送按钮：有内容时蓝色填充，无内容时禁用（灰色半透明）
- 禁止使用浮动按钮，输入框固定在底部

### 3.5 按钮

```
类型          样式                    使用场景
主要按钮      填充主色，文字白色          发送、确认、保存
次要按钮      填充透明，边框主色          取消、重试、查看更多
文字按钮      无背景无边框，主色文字      编辑、删除、次要操作
```

**通用规范**：
- 高度：44pt（iOS 最小触控目标）
- 内边距：水平 20pt，垂直 12pt
- 圆角：20pt
- 禁用态：opacity 0.4
- 点击反馈：`InkWell` + `HapticFeedback.lightImpact`

## 四、交互与动效

### 4.1 动画原则

```
原则 1：动画服务于理解，不为动而动
原则 2：时长统一，建立节奏感
原则 3：尊重系统设置（减少动态效果用户应禁用动画）
原则 4：即时反馈，不能让用户猜"点到了吗"
```

### 4.2 标准动画参数

| 场景 | 时长 | 曲线 | 说明 |
|------|------|------|------|
| 按钮点击反馈 | 100ms | easeOut | 即时 tactile 反馈 |
| 消息气泡入场 | 200ms | easeInOut | 从下方淡入上移 |
| 页面切换 | 300ms | easeInOut | 标准转场 |
| Sheet 弹出 | 350ms | easeOut | iOS 原生弹簧感 |
| 任务状态变化 | 200ms | easeInOut | 颜色/图标的过渡 |
| 骨架屏闪烁 | 1500ms | linear | 无限循环 |

**曲线速查**：
- `easeOut`：元素出现（进入屏幕）
- `easeIn`：元素消失（离开屏幕）
- `easeInOut`：状态变化（颜色、位置、大小）
- 弹簧曲线：仅用于 Sheet/Dialog 弹出

### 4.3 触觉反馈

```
场景                      反馈类型
消息发送成功              light
任务状态切换              medium
操作确认（保存/删除）     heavy
错误/警告                 notificationError（平台原生）
长按菜单                  light
滑动操作                  medium
```

### 4.4 骨架屏

AI 思考/工具调用期间显示骨架屏：

```
┌─────────────────────────┐
│ ░░░░░░░░ 3行 × 20pt     │  ← 灰色脉冲块
│ ░░░░░░░░ 2行 × 16pt     │
└─────────────────────────┘
```

**规则**：
- 仅在首次加载或长任务等待时使用
- 禁止与 Loading Spinner 同时出现
- 脉冲周期 1.5s，使用 `LinearProgressIndicator` 的 `min/max` 模拟
- 工具调用返回后立即替换为真实内容，不允许骨架屏停留超过 3s 无响应

## 五、深色模式

### 5.1 自动适配规则

```
浅色模式:
  背景: #F2F2F7
  卡片: #FFFFFF
  主文字: #000000
  次要文字: rgba(0,0,0,0.55)

深色模式:
  背景: #000000
  卡片: rgba(28,28,30,0.95)
  主文字: #FFFFFF
  次要文字: rgba(255,255,255,0.55)
```

### 5.2 强制规则

- 所有颜色必须使用语义化命名（`surfacePrimary`、`textSecondary`），禁止硬编码 `Colors.white`
- 渐变在深色模式下降低饱和度 20%
- 阴影在深色模式下使用 `rgba(255,255,255,0.1)` 代替黑色阴影

## 六、无障碍

### 6.1 必做清单

- [ ] 所有可交互元素 ≥ 44 × 44pt
- [ ] 颜色对比度 ≥ 4.5:1（WCAG AA）
- [ ] 支持系统字体缩放（动态字号）
- [ ] 所有图标配有 `Semantics` 标签
- [ ] AI 回复支持系统读屏（`Semantics` 包裹文本）
- [ ] 错误提示配有语音提示

### 6.2 AI 回复的无障碍处理

```
问题：AI 回复中可能包含工具状态、代码块、按钮
解决：
- 代码块使用 Code Block Semantics，读屏时读"代码块"
- 状态变化使用 LiveRegion，读屏自动播报
- 链接和按钮必须设置 semanticLabel
```

## 七、性能与稳定性

### 7.1 流畅度底线

```
60fps 是底线，不是目标
列表滚动必须保持 60fps
AI 流式输出时禁止阻塞 UI 线程
图片懒加载，超出视口 200px 开始预加载
```

### 7.2 加载状态规范

```
场景                      展示方式
网络请求 < 300ms          不展示 loading（直接返回）
网络请求 300ms - 2s       骨架屏
网络请求 > 2s             骨架屏 + "正在处理中..." 文字
工具调用                  内联状态条（见 3.2）
长时间生成（> 10s）       进度条 + 预计剩余时间（如有）
```

### 7.3 错误恢复

```
错误类型          用户看到                          操作
网络中断          "网络似乎不太稳定" + 重试按钮      自动重试 1 次
工具调用失败      "遇到点问题：xxx" + 重试按钮       不阻断流程
AI 生成失败       灰色占位 + "内容加载失败" + 重试    保持上下文
未知错误          "出错了，请稍后再试"              日志上报
```

**规则**：所有错误必须提供**可操作的下一步**，禁止只显示错误不提供解决方案。

## 八、内容策略（AI 对话专用）

### 8.1 AI 回复的语气

```
✅ 简洁、具体、可操作
❌ 冗长铺垫、"当然可以"、"很高兴为您服务"
✅ 直接给出答案，必要时附说明
❌ 反问用户"您需要我帮您吗？"（用户已经问了）
```

### 8.2 消息长度

```
普通对话：1-3 屏可读完（约 300 字以内）
长回复：分段 + 摘要开头
代码输出：附带运行说明
列表/步骤：每步 1 行，不超过 7 步
```

### 8.3 工具调用的用户感知

```
不要显示原始 JSON/参数
只展示：正在做什么 + 结果摘要
例：
  ❌ 调用 weather(lat=39.9, lon=116.4, unit=celsius)
  ✅ 正在查询北京天气... 今天 26°C，晴
```

## 十、扩展规则

### 10.1 新增工具时的 UI 规范

当新增任何工具调用时，必须配套：
1. 状态条样式（进行中/成功/失败）
2. 结果展示方式（文本/卡片/列表）
3. 错误状态文案

### 10.2 多 Agent 协调模式下的 UI

当 Manager 模式启用时：
- 当前发言者名称显示在消息顶部，字体 Caption 2
- 多个 Agent 的消息用左侧彩色条区分（每个 Agent 固定一种颜色）
- 切换 Agent 时有一个 150ms 的淡入淡出

### 10.3 未来适配

```
Vision Pro 空间计算：
- 消息气泡悬浮在空间中
- 深度感通过阴影层级实现
- 手势交互：捏合确认、悬停预览
```
