# DWeis AI 助手 — UI 重设计方案（极简对话式）

> 设计方向：现代极简对话式（Claude / ChatGPT 用户心智）
> 设计原则：**升级而非换血** —— 延续品牌蓝 `#007AFF`，保留用户已习惯的交互，只重做 App 框架层与统一设计令牌。
> 关键约束：**AI（助手）消息气泡保持现状不变**（无气泡纯文本流 + 步骤状态行，见 §2.4）。

---

## 0. 范围界定

| 层 | 本次处理 | 说明 |
|---|---|---|
| 消息呈现层（AI / 用户气泡） | **不改** | AI 无气泡纯文本流、用户蓝填充圆角气泡均为用户习惯，保留 |
| App 框架层 | **重做** | 导航、首页/会话列表、设置、空状态、顶栏、侧边栏 |
| 设计令牌 | **重组** | 在现有 `AgentColors` 基础上扩展为极简对话式令牌体系 |
| 组件库 | **补齐规范** | 按钮 / 输入 / 卡片 / 导航 / 反馈 / 状态统一规格 |

---

## 1. 设计基础（Design Tokens）

### 1.1 颜色系统

基于现有 `AgentColors` 重组，强调近白底 + 克制留白 + 单一品牌蓝。

| Token | 浅色值 | 用途 | 对比度 |
|---|---|---|---|
| `--bg-base` | `#FFFFFF` | 主背景（页面/卡片） | — |
| `--bg-subtle` | `#F7F7F8` | 次级区（输入框底、助手区、hover） | — |
| `--bg-elevated` | `#FFFFFF` | 浮层/弹窗/抽屉 | — |
| `--text-primary` | `#1C1C1E` | 主文字 | 16:1 on `#FFFFFF` ✅ |
| `--text-secondary` | `#8A8A8E` | 次级文字/预览 | 4.6:1 ✅ |
| `--text-tertiary` | `#C7C7CC` | 提示/hint | 2.3:1（仅大号/非正文） |
| `--brand` | `#007AFF` | 品牌主色（按钮/激活/链接） | 4.7:1 on white ✅ |
| `--brand-hover` | `#0056CC` | 按压/ hover | 6.9:1 ✅ |
| `--brand-soft` | `#E6F1FB` | 选中/高亮底（淡蓝） | — |
| `--success` | `#34C759` | 成功 | — |
| `--warning` | `#FF9500` | 警告 | — |
| `--error` | `#FF3B30` | 错误/删除 | — |
| `--border` | `#E5E5EA` | 0.5px 分隔线/描边 | — |

> 全部文字色对比度满足 WCAG AA（正文 ≥ 4.5:1）。`--text-tertiary` 仅用于非正文大号提示。

### 1.2 字体系统

- 字体栈：系统无衬线 `system-ui / -apple-system / "Segoe UI" / Roboto / "PingFang SC" / "Microsoft YaHei"`
- 字重：仅 **400（常规）/ 500（中粗）** 两档，保持极简克制
- 字号阶梯（基于 4pt 栅格与 1.2 行高比）：

| 角色 | 字号 | 字重 | 行高 | 示例 |
|---|---|---|---|---|
| 大标题 | 20 / 28px | 500 | 1.3 | 页面主标题 |
| 标题 | 17px | 500 | 1.35 | 卡片标题 / 会话名 |
| 正文 | 15px | 400 | 1.5 | 用户气泡 / 列表预览 |
| 次要 | 13px | 400 | 1.5 | 步骤状态 / 说明 |
| 辅助 | 12px | 400 | 1.4 | 时间 / 标签 |
| 微型 | 11px | 400 | 1.3 | Tab 角标 |

### 1.3 间距系统（4pt 栅格）

基础单位 4px。统一节奏：

`4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 / 64`

- 组件内边距：12–16px
- 列表行间距：16px（行高 64–72px）
- 区块间距：24–32px
- 页面左右安全边距：16px（手机）

### 1.4 圆角

| Token | 值 | 用途 |
|---|---|---|
| `--radius-sm` | 8px | 小标签 / chip |
| `--radius-md` | 12px | 卡片 / 按钮 / 输入框 |
| `--radius-lg` | 16px | 底部 Sheet / Modal 顶部 |
| `--radius-bubble` | 18px | 用户消息气泡（现状保留） |
| `--radius-pill` | 22px | 底部输入胶囊 / 头像组 |

### 1.5 阴影与深度（极克制）

极简对话式**几乎不用阴影**，靠 0.5px 边框 + 留白建立层级：

- 默认：无阴影，仅 `--border` 描边
- 浮层（Sheet / Modal / 抽屉）：`0 8px 24px rgba(0,0,0,0.12)`，仅出现在层级最高的临时层
- 按压反馈：背景色微变（非位移/阴影）

### 1.6 动效

- 时长：微交互 150ms，页面/浮层 250ms
- 曲线：`ease-out`
- 尊重 `prefers-reduced-motion`：关闭非必要位移动画
- 消息流式：现有 `FadeTransition`（0.55→1.0, 200ms）保留

## 2. 组件库

### 2.1 按钮

| 变体 | 样式 | 用途 |
|---|---|---|
| Primary | `--brand` 填充 + 白字，radius-md，高 40px | 主操作（发送/确认） |
| Secondary | `--bg-subtle` 底 + `--text-primary` 字 + 0.5px 边 | 次操作 |
| Ghost | 透明 + `--text-secondary` 字 | 工具栏图标按钮 |

- 触控目标 ≥ 44×44px
- 禁用态：opacity 0.5
- Focus：2px `--brand` 外环（键盘可达）

### 2.2 输入框（底部常驻胶囊）

- 形态：底部固定胶囊，`radius-pill`，`--bg-subtle` 底 + 0.5px 边
- 占位文字 `--text-tertiary`，输入文字 `--text-primary` 15px
- 右侧发送键：圆形 `--brand` 底 + 白箭头（加载时变停止图标）
- 支持 @提及触发、附件、图片（现状能力保留）

### 2.3 卡片

- 白底，`--border` 0.5px 描边，radius-md，内边距 16px
- 无阴影（层级靠间距，非投影）
- 列表行可不用独立卡片，用 0.5px 分隔线 + 留白

### 2.4 消息气泡 ⚠️ 关键约束

**AI（助手）消息 — 保持现状不变：**
- 无气泡背景，纯文本左对齐（`crossAxisAlignment: start`）
- 带流式渐显动画 + "思考中 / 工具调用"步骤状态行（现状 `_AIBubble`）
- Markdown 渲染（`buildInlineContent`）保留
- 理由：这本身就是 Claude/ChatGPT 式极简范式，且用户明确要求不改

**用户消息 — 保持现状：**
- `--brand`（`#007AFF`）填充 + 白字，radius-bubble 18px，右对齐
- maxWidth 300，内边距 14×10
- 图片/文档附件卡片沿用现状蓝色容器

### 2.5 导航

**底部三段 Tab**（主页/探索/库）
- 激活态 `--brand`，未激活 `--text-secondary`
- Tab 高度 56px，图标 22px + 文字 11px
- 顶部分隔线 0.5px

**左侧抽屉（侧边栏）**
- 已改为**不全屏**：宽度 = 屏宽 − 56dp（最大 320dp），右侧露出聊天内容
- 含：会话/分组入口、Skill & MCP（puzzlePiece）、设置（gear）等
- 半透明遮罩点击关闭

**顶栏（AppBar）**
- 已收矮：工具栏高 48px，毛玻璃 `BackdropFilter`，**无底部阴影线**
- 左：菜单图标（20px）；中/左：标题；右：上下文/模型入口

### 2.6 反馈

- Toast：底部上浮，白底 0.5px 边 + 轻阴影，2s 自动消失
- Bottom Sheet：顶部 radius-lg 16px，顶部把手条（36×4 圆角）
- Modal：居中卡片，radius-md

### 2.7 状态

- 骨架屏：`-subtle` 底色块 + 微光（shimmer），替代 spinner
- 空状态：居中插画（线性图标）+ 一句引导文案 + 主操作按钮
- 加载中：消息区显示"思考中"扫光（现状保留）

## 3. 主要界面规格

### 3.1 首页 / 会话列表（见上方 mockup）
- 顶栏：标题"对话" + 右侧搜索 / 新建
- 列表行：圆形头像（18px 半径，首字母/色块）+ 标题 15px + 预览 12px 次级 + 时间 11px 右对齐
- 行高 64–72px，0.5px 分隔线，点击进会话
- 底部三段 Tab，主页激活为蓝

### 3.2 单聊（见上方 mockup）
- 顶栏：菜单 + 标题（Agent 名）+ 头像
- 消息流：AI 无气泡纯文本 / 用户蓝气泡（现状）
- 底部胶囊输入常驻

### 3.3 群聊（多 Agent 协作）
- 每条消息带 **Agent 身份徽标**（彩色头像 + 名字），区分发言者
- 支持 `@` 提及（底部输入框触发，弹出成员 sheet）
- 顶部状态栏：圆形状态徽标 + 参与人数 + 协作模式指示
- 其余沿用单聊框架，气泡区分色来自各 Agent 主题色

### 3.4 设置
- iOS 风分组列表：分组标题 12px 次级 + 白底行（左标签 + 右值/箭头）
- 大量留白，0.5px 分组分隔
- 入口：模型、Skill & MCP、上下文文档、关于

## 4. 暗色模式

映射现有 `AgentColors.dark()`：

| Token | 暗色值 |
|---|---|
| `--bg-base` | `#000000` |
| `--bg-subtle` | `#1C1C1E` |
| `--text-primary` | `#F2F2F7` |
| `--text-secondary` | `#8E8E93` |
| `--brand` | `#0A84FF` |
| `--border` | `rgba(235,235,245,0.2)` |

- 气泡：用户 `--brand` 填充不变；AI 纯文本在黑底用 `--text-primary`
- 对比度复核：暗色下 `--text-primary` on `#000` ≈ 17:1 ✅

---

## 5. 无障碍（WCAG AA）

- 文字对比度：正文 ≥ 4.5:1，大号 ≥ 3:1（全部满足）
- 触控目标：交互元素 ≥ 44×44px
- 键盘导航：所有操作可 Tab 到达，Focus 可见（2px 品牌环）
- 屏幕阅读器：语义标签（`AppBar`/`Drawer`/`ListTile` 现状已具），图标按钮加 `semanticsLabel`
- 动效敏感：尊重 `prefers-reduced-motion`
- 文字缩放：布局支持系统字体放大至 200% 不破版

## 6. 落地建议（映射现有代码）

1. **令牌**：在 `lib/core/agent_colors.dart` 现有字段上补充 `--bg-subtle` / `--brand-soft` / 圆角/间距常量（无需新建整套，避免碎片化）。
2. **组件规范**：按 §2 统一按钮/输入/卡片的 padding、圆角、字重，消除散落 magic number。
3. **框架层**：首页列表、设置分组、空状态按 §3 规格重做；顶栏/侧边栏已在前期微调中完成（收矮、不全屏、去阴影）。
4. **消息层**：§2.4 明确**不改**，保持现状。
5. **分阶段**：先令牌 + 组件，再首页/设置，最后群聊身份视觉；每阶段跑 `flutter analyze` + 现有 widget 测试回归。
